üèóÔ∏è System Architecture Components

1Broadcaster Client

Streamer's software for capturing and sending video to ingest servers

  • OBS Studio: Most popular (open source)
  • Streamlabs: OBS fork with integrations
  • XSplit: Professional streaming software
  • Native Apps: Twitch Studio, YouTube app
  • Mobile: iOS/Android streaming apps
  • Protocol: RTMP, RTMPS, SRT

2Ingest Servers

Receive live video streams from broadcasters globally

  • Protocol: RTMP (Real-Time Messaging Protocol)
  • Alternative: SRT (Secure Reliable Transport)
  • WebRTC: For browser-based streaming
  • Global PoPs: 100+ ingest locations
  • Redundancy: Dual ingest (primary + backup)
  • Tools: Nginx-RTMP, Wowza, AWS MediaLive

3Real-time Transcoding

Convert live stream to multiple bitrates on-the-fly

  • Input: 1080p 60fps @ 6 Mbps (typical)
  • Outputs: 1080p60, 720p60, 720p, 480p, 360p, 160p
  • Latency Added: 2-5 seconds
  • Encoder: Hardware (NVIDIA GPU), FFmpeg
  • Codec: H.264 (primary), VP9, AV1
  • Audio: AAC, Opus

4Stream Distribution

Deliver live streams to millions of viewers

  • HLS (Standard): 6-10s segments, 20-30s latency
  • Low-Latency HLS: 1-2s segments, 3-8s latency
  • LL-DASH: Chunked transfer, 2-5s latency
  • WebRTC: Sub-second latency (<1s)
  • CMAF: Common Media Application Format
  • Twitch: Uses modified HLS protocol

5CDN Layer

Global edge network for video delivery

  • Twitch: Amazon CloudFront + custom edges
  • YouTube Live: Google Global Cache
  • Edge Locations: 200+ worldwide PoPs
  • Origin Shield: Protect ingest servers
  • Cache: Live segments (short TTL: 30s-2min)
  • Fallback: Multi-CDN for redundancy

6Chat System

Real-time messaging for viewer interaction

  • Protocol: IRC (Twitch), WebSocket
  • Architecture: Elixir/Phoenix (Twitch)
  • Sharding: By channel/room
  • Rate Limiting: Per user (20 msg/30s)
  • Moderation: AutoMod, ban lists, slow mode
  • Emotes: Custom images, GIFs, badges
  • Scale: 15+ billion messages/day (Twitch)

7Stream Metadata Service

Manage stream info, status, and discovery

  • Database: PostgreSQL (channels, users)
  • NoSQL: DynamoDB (stream state, viewers)
  • Redis: Real-time viewer count
  • Fields: Title, game, tags, category
  • Status: Live, offline, VOD processing
  • Discovery: Elasticsearch for search

8VOD (Video on Demand)

Archive streams for later viewing

  • DVR: Record while streaming
  • Storage: AWS S3, Google Cloud Storage
  • Processing: Generate highlights, clips
  • Retention: 14-60 days (varies by tier)
  • Clips: User-generated 5-60s highlights
  • Chapters: Automatic stream segmentation

9Real-time Analytics

Live metrics and viewer engagement tracking

  • Viewer Count: Real-time (Redis + WebSocket)
  • Bitrate/Quality: Per-viewer ABR tracking
  • Buffering Events: QoE monitoring
  • Chat Activity: Messages/second
  • Engagement: Follows, subs, bits, donations
  • Pipeline: Kinesis ‚Üí Lambda ‚Üí DynamoDB

10Monetization Engine

Revenue systems for streamers and platform

  • Subscriptions: Monthly recurring ($4.99-$24.99)
  • Bits/Cheers: Virtual currency for tips
  • Ads: Pre-roll, mid-roll (programmatic)
  • Donations: Third-party (StreamLabs, Patreon)
  • Sponsorships: Brand deals, affiliate links
  • Processing: Stripe, PayPal integration

11Recommendation System

Surface relevant live streams to viewers

  • Browse Page: Categorized by game/topic
  • Personalized: Based on watch history
  • Trending: Highest viewer growth rate
  • Following: Channels user follows (priority)
  • Discovery: Similar channels, new streamers
  • Algorithm: Collaborative filtering + ML

12Content Moderation

Automated and manual content filtering

  • AutoMod: ML-based chat filtering
  • Video Screening: Real-time content analysis
  • DMCA: Copyright detection (audio fingerprinting)
  • Reports: User flagging system
  • Human Review: Moderation team escalation
  • Actions: Warnings, timeouts, bans

13Authentication & Auth

User authentication and authorization

  • OAuth 2.0: Login with Twitch/Google/Facebook
  • JWT Tokens: Session management
  • Stream Keys: Broadcaster authentication
  • Permissions: Streamer, mod, VIP, subscriber
  • 2FA: Two-factor authentication
  • API Keys: Third-party integrations

14Notifications System

Alert viewers when streams go live

  • Push Notifications: Mobile (FCM, APNs)
  • Email: Batch notifications (SES)
  • Discord Webhooks: Third-party integrations
  • In-app: Browser notifications
  • Preferences: Per-channel notification settings
  • Batching: Aggregate multiple go-live events

15Extensions & Overlays

Interactive features and broadcaster tools

  • Twitch Extensions: iFrame-based panels
  • Overlays: Alerts, donation goals, chat on stream
  • Widgets: StreamLabs, StreamElements
  • Polls: Real-time viewer voting
  • Predictions: Channel points betting
  • API: Third-party developer platform

16Multi-streaming

Broadcast to multiple platforms simultaneously

  • Restream.io: Stream to 30+ platforms
  • Architecture: Receive once, fanout to N platforms
  • Platforms: Twitch, YouTube, Facebook, Twitter
  • Chat Aggregation: Unified chat from all platforms
  • Analytics: Cross-platform metrics
