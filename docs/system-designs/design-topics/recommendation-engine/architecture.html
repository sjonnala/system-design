ü§ñ ML MODEL ARCHITECTURE DETAILS

Two-Tower Neural Network

Architecture:
‚Ä¢ User Tower: Embedding(1B users, 64) ‚Üí Dense(128) ‚Üí Dense(128)
‚Ä¢ Item Tower: Embedding(100M items, 64) ‚Üí Dense(128) ‚Üí Dense(128)
‚Ä¢ Similarity: Dot product of user/item vectors

Training: Sampled softmax loss, negative sampling ratio 1:100
Serving: Pre-compute item embeddings, ANN search for candidates

XGBoost Ranking Model

Features (200+ features):
‚Ä¢ User: demographics, behavior stats, embeddings
‚Ä¢ Item: popularity, CTR, category, embeddings
‚Ä¢ User-Item: affinity score, similarity
‚Ä¢ Context: time, device, location

Training: LambdaRank objective (learning-to-rank)
Inference: ~5ms for scoring 500 candidates

Matrix Factorization (ALS)

Algorithm: Alternating Least Squares
‚Ä¢ Latent factors: 128 dimensions
‚Ä¢ Regularization: L2 (Œª = 0.01)
‚Ä¢ Implicit feedback (clicks, views)

Use Case: Baseline collaborative filtering
Update Frequency: Daily batch retraining

Sequential Transformer (BST)

Architecture: Behavior Sequence Transformer
‚Ä¢ Input: Last 50 user interactions
‚Ä¢ Multi-head attention (8 heads, 128 dims)
‚Ä¢ Position encoding for temporal patterns

Advantage: Captures short-term user intent
Latency: ~20ms inference time

üé® SYSTEM DESIGN PRINCIPLES

Scalability Strategies

  • Horizontal Scaling: Stateless services, load balancing
  • Caching: Multi-tier (Redis, CDN, client-side)
  • Sharding: User-based sharding for interactions
  • Async Processing: Event-driven architecture (Kafka)
  • Pre-computation: Batch-generated recommendations

ML Engineering Best Practices

  • Feature Store: Centralized, versioned features
  • Model Versioning: A/B test multiple models
  • Online Learning: Continuous model updates
  • Monitoring: Track model drift, data quality
  • Fallback: Multiple model candidates

Data Consistency

  • Eventual Consistency: User interactions (acceptable delay)
  • Strong Consistency: Purchases, ratings (critical)
  • CQRS: Separate read/write models
  • Event Sourcing: Rebuild state from events

Fault Tolerance

  • Graceful Degradation: Fallback to trending/popular
  • Circuit Breakers: Prevent cascading failures
  • Retry Logic: Exponential backoff
  • Multi-Region: Geographic redundancy
