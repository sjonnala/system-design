üöï Ride Request Flow

Step 1: Rider Requests Ride

Client Action: User opens app, selects pickup & dropoff locations

POST /api/v1/rides/request { "pickup": {"lat": 37.7749, "lon": -122.4194, "address": "123 Market St"}, "dropoff": {"lat": 37.7849, "lon": -122.4094, "address": "456 Mission St"}, "car_type": "uberx", "payment_method": "card_ending_1234" }

Backend: API Gateway validates token, checks rate limit ‚Üí forwards to Ride Service

Step 2: Calculate Fare Estimate

Pricing Service: Calculates fare based on:

  • Distance (Google Maps API)
  • Estimated time (traffic data)
  • Base fare + per mile/minute rates
  • Surge multiplier (if high demand)
base_fare = 2.50 distance_fare = distance_km * 1.50 time_fare = duration_min * 0.25 surge_multiplier = calculate_surge(pickup_location) estimated_fare = (base_fare + distance_fare + time_fare) * surge_multiplier

Step 3: Find Nearby Drivers

Location Service: Query Redis Geo index

GEORADIUS drivers:online -122.4194 37.7749 5 km WITHDIST ASC ‚Üí Returns: [driver_123, driver_456, driver_789]

Filter: car_type=uberx, rating>4.5, status=available

Step 4: Create Trip & Match Driver

Matching Service: Triggers driver matching algorithm

  • Calculate ETA for each candidate driver
  • Score drivers (distance, ETA, rating, acceptance rate)
  • Dispatch to top 3 drivers in parallel
  • First to accept wins

Step 5: Response to Rider

Response: 201 Created { "trip_id": "trip_abc123", "state": "REQUESTED", "estimated_fare": { "base": 8.50, "surge_multiplier": 1.5, "total": 12.75 }, "estimated_wait_time": 3, // minutes "nearby_drivers_count": 5 }

üéØ Driver Matching Flow

Step 1: Get Nearby Drivers

# Redis Geo query (5km radius) candidates = GEORADIUS drivers:online {pickup_lon} {pickup_lat} 5 km WITHDIST ASC # Result: [ # {driver_id: 'driver_123', distance: 0.8}, # {driver_id: 'driver_456', distance: 1.2}, # {driver_id: 'driver_789', distance: 2.5} # ]

Step 2: Filter Candidates

Criteria:

  • Car type matches request (uberx, uberxl, etc.)
  • Driver rating ‚â• 4.5 stars
  • Driver is available (not on another trip)
  • Driver acceptance rate > 80%

Step 3: Calculate ETA for Each Driver

for driver in candidates: # Get route from driver location to pickup route = google_maps.get_directions( origin=driver.location, destination=pickup_location, mode='driving', traffic_model='best_guess' ) driver.eta = route.duration_in_traffic driver.route_polyline = route.overview_polyline

Step 4: Score & Rank Drivers

for driver in candidates: score = ( 100 / (1 + driver.distance) + # 40% weight 50 / (1 + driver.eta / 60) + # 25% weight (ETA in minutes) driver.rating * 10 + # 25% weight driver.acceptance_rate * 10 + # 10% weight (10 if is_heading_towards(driver, pickup) else 0) # Bonus ) driver.match_score = score candidates.sort(key=lambda d: d.match_score, reverse=True)

Step 5: Dispatch to Top Drivers (Parallel)

Send ride request to top 3 drivers simultaneously

top_drivers = candidates[:3] for driver in top_drivers: # Send push notification fcm.send({ to: driver.fcm_token, priority: 'high', notification: { title: 'New Ride Request', body: f'{eta} min away ‚Ä¢ ${fare} fare' }, data: { trip_id, pickup_lat, pickup_lon, dropoff_lat, dropoff_lon, fare } }) # Wait for first acceptance (15s timeout) accepted_driver = await wait_for_acceptance(trip_id, timeout=15) if accepted_driver: # Cancel other requests for d in top_drivers: if d.id != accepted_driver.id: cancel_request(d.id, trip_id)

üõ£Ô∏è Trip Lifecycle Flow

State Transition Diagram

REQUESTED ‚Üì (driver accepts) DRIVER_ASSIGNED ‚Üì (driver arrives at pickup) DRIVER_ARRIVED ‚Üì (rider gets in car, driver taps "Start Trip") TRIP_STARTED ‚Üì (destination reached, driver taps "End Trip") TRIP_COMPLETED ‚Üì (payment processed) PAYMENT_PROCESSED ‚Üì (both parties rate each other) RATED

State: DRIVER_ASSIGNED

Actions:

  • Update trip state in PostgreSQL & Redis
  • Send push notification to rider (driver details)
  • Start ETA tracking (update every 10 seconds)
  • Enable real-time location tracking for rider
db.update_trip(trip_id, { state: 'DRIVER_ASSIGNED', driver_id: driver_id, assigned_at: now() }) websocket.send_to_rider(rider_id, { event: 'driver_assigned', driver: { name, photo, rating, car_make, car_model, license_plate }, eta_minutes: 3 })

State: TRIP_STARTED

Actions:

  • Start fare meter (distance + time tracking)
  • Record trip route (polyline for receipt)
  • Update both apps with "Trip in Progress"
# Start fare calculation trip.start_time = now() trip.start_location = driver.current_location # Track route in background setInterval(() => { trip.route_polyline.append(driver.current_location) trip.distance_traveled += distance_since_last_update() }, 10000) # Every 10 seconds

State: TRIP_COMPLETED ‚Üí Payment

Calculate Final Fare:

base_fare = 2.50 distance_fare = trip.distance_traveled_km * 1.50 // per km time_fare = trip.duration_minutes * 0.25 // per minute surge_multiplier = trip.surge_multiplier tip = rider_selected_tip final_fare = (base_fare + distance_fare + time_fare) * surge_multiplier + tip # Process payment via Stripe payment_result = stripe.charge.create({ amount: final_fare * 100, // cents currency: 'usd', customer: rider.stripe_customer_id, idempotency_key: f'trip:{trip_id}', metadata: {trip_id, rider_id, driver_id} }) # Update trip & notify both parties db.update_trip(trip_id, { state: 'PAYMENT_PROCESSED', final_fare: final_fare, payment_id: payment_result.id })
