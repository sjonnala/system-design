üî¢ ID GENERATION
Distributed ID Generator
Strategy: Base62 Encoding
// Python implementation import hashlib import base64 BASE62 = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ" def generate_short_code(url, counter): hash_input = f"{url}{counter}{timestamp}" hash_value = hashlib.md5( hash_input.encode() ).digest() # Convert to base62 num = int.from_bytes(hash_value[:6]) code = "" while num > 0: code = BASE62[num % 62] + code num //= 62 return code[:7] # 7 chars = 62^7 = 3.5 trillion URLs
Alternatives:
  • Snowflake ID: 64-bit timestamp + machine ID + sequence
  • UUID: Collision-resistant but longer
  • Counter + Base62: Simple, sequential

üîê ADDITIONAL SYSTEM CONSIDERATIONS

Security Measures

  • URL Validation: Blacklist malicious domains, malware scanning
  • Rate Limiting: Per IP, per user, per API key
  • DDoS Protection: CloudFlare, AWS Shield
  • CAPTCHA: For suspicious traffic patterns
  • HTTPS Only: TLS 1.3, HSTS headers
  • API Authentication: JWT tokens, API keys with rotation

Scalability Strategies

  • Horizontal Scaling: Auto-scaling groups (2-50 instances)
  • Database Sharding: Consistent hashing by short_code
  • Read-Write Separation: CQRS pattern
  • Caching Strategy: Multi-tier (CDN, Redis, Application)
  • Async Processing: Event-driven for analytics
  • Geographic Distribution: Multi-region deployment

Data Consistency

  • Write: Strong consistency (PostgreSQL ACID)
  • Read: Eventual consistency (replica lag <100ms)
  • Cache Invalidation: TTL-based + event-driven
  • Conflict Resolution: Last-write-wins for duplicates

Failure Scenarios & Recovery

  • Database Failover: Auto-promote replica (30s RTO)
  • Cache Failure: Fallback to database (circuit breaker)
  • Service Failure: Health checks + auto-restart
  • Region Failure: DNS failover to backup region (5min RTO)
  • Data Loss: Point-in-time recovery, daily backups
