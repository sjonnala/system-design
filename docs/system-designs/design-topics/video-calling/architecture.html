üé¨ SFU (MEDIA SERVER)
Selective Forwarding Unit
Architecture:
// SFU vs MCU vs P2P P2P (Peer-to-Peer): ‚úì Low latency, no server cost ‚úó Scales poorly (n¬≤ connections) Use: 1-on-1 calls MCU (Multipoint Control Unit): ‚úì Low client bandwidth ‚úó High server CPU (mixing/encoding) Use: Large conferences (100+) SFU (Selective Forwarding): ‚úì Medium server cost ‚úì Good scalability (10-50 users) ‚úì Low latency Use: Most group calls (Zoom default)
SFU Responsibilities:
  • Forward video/audio streams (no mixing)
  • Simulcast handling (multiple qualities)
  • Bandwidth estimation & adaptation
  • Selective forwarding (send only visible streams)
  • Server-side bandwidth control
Popular SFU Implementations:
  • Mediasoup (Node.js) - Used by Whereby
  • Janus (C) - High performance
  • Jitsi Videobridge (Java) - Jitsi Meet
  • Pion (Go) - Cloud-native
  • Zoom: Custom C++ SFU

üîê ADDITIONAL SYSTEM CONSIDERATIONS

Security & Encryption

  • Transport Security: DTLS-SRTP for media, TLS for signaling
  • End-to-End Encryption: Optional E2EE (Zoom, Meet)
  • Authentication: OAuth 2.0, SSO for enterprise
  • Meeting Security: Passwords, waiting rooms, host controls
  • Data Encryption: At-rest encryption for recordings
  • Compliance: HIPAA, SOC 2, GDPR

Scalability Strategies

  • SFU Clustering: Route by geography, load balance
  • Cascading SFUs: Large meetings (100+ participants)
  • Auto-scaling: Spin up SFUs based on demand
  • Regional Deployment: Multi-region for low latency
  • Simulcast: Multiple quality tiers per stream
  • VP9 SVC: Scalable Video Coding for bandwidth

Quality of Experience (QoE)

  • Adaptive Bitrate: Adjust to network conditions
  • Audio Priority: Maintain audio quality first
  • Jitter Buffer: Smooth out network variations
  • Packet Loss Recovery: FEC (Forward Error Correction)
  • Echo Cancellation: AEC algorithms
  • Noise Suppression: ML-based background noise removal

Failure Scenarios & Recovery

  • SFU Failure: Migrate participants to backup SFU (<30s)
  • Network Issues: Automatic quality degradation
  • TURN Fallback: When P2P/STUN fails
  • Signaling Failure: WebSocket reconnection
  • Recording Failure: Retry + notification to host
  • Multi-Region: Failover to nearest region
