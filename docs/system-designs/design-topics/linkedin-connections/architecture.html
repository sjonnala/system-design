üï∏Ô∏è GRAPH DATABASE ARCHITECTURE

Neo4j Schema

// Nodes
(:User {userId, name, email, ...})
(:Company {companyId, name, ...})
(:School {schoolId, name, ...})

// Relationships
(User)-[:CONNECTED_TO]->(User)
(User)-[:WORKS_AT]->(Company)
(User)-[:ATTENDED]->(School)
(User)-[:REQUESTED]->(User)
(User)-[:BLOCKED]->(User)

Query Examples

// Find mutual connections
MATCH (a:User)-[:CONNECTED]-(m)-[:CONNECTED]-(b:User)
WHERE a.userId = $userA AND b.userId = $userB
RETURN m

// Calculate degree
MATCH path = shortestPath(
  (a:User)-[:CONNECTED*1..3]-(b:User)
)
RETURN length(path)

Sharding Strategy

Shard by User ID:
- Hash user_id % 16 = shard_id
- Each shard: 50M users
- 3x replication per shard
- Cross-shard queries via coordinator

Trade-off: Most connections are within shard (80%+), minimizing cross-shard queries
